# 学生成绩管理系统 · 用户使用说明文档（教师 / 院长 / 管理员 / 学生用）

**版本：** v2.0  
**更新日期：** 2026-01-31  
**适用对象：** 教师、院长、系统管理员、学生  

---

## 一、登录方式

### 1.1 登录入口与账号

- **登录地址**：部署后为 `http://服务器地址/`（如 http://192.168.3.108 或 http://localhost:8080）。
- **账号与密码**：  
  - **学生**：使用**学号** + 密码（如学号 `S240101`，初始密码 `123`）。  
  - **教师 / 院长 / 管理员**：使用**工号** + 密码（如工号 `T1001`、`admin`、`dean1`，初始密码 `123`）。  
- **不支持使用数字 ID 登录**，必须使用学号或工号。

### 1.2 角色选择与限制

登录前需在页面上选择**用户类型**：

| 用户类型 | 说明 | 登录后进入 |
|----------|------|------------|
| 学生 | 学生本人 | 学生端首页 |
| 老师 | 普通教师 | 教师端首页 |
| 院长 | 学院院长 | 院长端首页 |
| 管理员 | 系统管理员 | 管理端首页 |

**限制说明：**

- 学号只能选择「学生」类型登录；工号只能选择「老师」「院长」「管理员」之一，且系统会校验工号对应的实际角色是否与选择一致，不一致会提示「角色不匹配」。
- 首次使用建议由管理员告知学号/工号及初始密码，登录后及时修改密码（若系统已开放修改密码功能）。

## 二、成绩单上传

### 2.1 谁可以上传、上传范围

| 角色 | 上传范围 | 说明 |
|------|----------|------|
| **教师** | 仅本人任课课程 | 系统按当前登录教师自动限定「本人任课」的课程，无需选课程，只需选学院（自动为本院）。 |
| **院长** | 仅本学院课程 | 系统按当前登录院长自动限定「本学院」课程，上传时无需再选学院。 |
| **管理员** | 全部课程 | 上传前需在页面上**选择学院**，然后上传该学院下某门课的成绩单。 |

### 2.2 文件格式要求

- **格式**：Excel 文件，支持 `.xlsx` 或 `.xls`。
- **结构**：成绩单为**固定格式**，不能随意改表头：
  - **第 2 行**（合并单元格）：课程名称、课程类别、课程性质、考核方式、任课教师。
  - **第 3 行**（合并单元格）：开课学期、学分、学时、开课班级。
  - **第 5 行起**：学号、姓名、平时成绩、期中成绩、期末成绩、总成绩、备注等。

示例（第 2 行）：

```
课程名称：临床检验基础  课程类别：专业课  课程性质：必修  考核方式：考试  任课教师：李老师
```

示例（第 3 行）：

```
开课学期：2024-2025-1  学分：4  学时：64  开课班级：24级医学检验技术1班
```

- **重要**：课程名称、开课学期、任课教师、开课班级等必须与系统中已维护的**完全一致**（含标点、空格），否则整单会被拒绝。

### 2.3 上传前准备

1. 确认**学期、课程、任课教师、开课信息**已在系统中维护（管理员在「教学管理」中维护开课表）。
2. 确认**学生**已导入，且 Excel 中的学号与系统中学号一致。
3. 同一学生、同一课程、同一学期若已有成绩，不能再次上传覆盖，需走「成绩修改申请」流程。

## 三、下载功能

### 3.1 下载某个学生所有学期成绩单

- **适用角色**：管理员、院长、教师（按权限看到相应学生）。
- **操作**：在「成绩管理」相关页面中，按学生、学期等条件查询后，使用「导出」或「下载成绩单」类功能（若已提供），可得到该学生各学期成绩的汇总或分学期成绩单。
- **说明**：具体菜单位置以系统实际菜单为准，一般位于成绩查询/统计相关页面。

### 3.2 下载某学期挂科学生列表

- **适用角色**：管理员、院长、教师（按权限）。
- **操作**：在成绩查询中筛选「某学期」「总成绩低于 60 或未通过」，再使用导出功能，得到该学期挂科学生列表。
- **说明**：若系统提供「挂科名单导出」或「补考名单导出」，可直接使用。

### 3.3 下载历史导入的成绩单文件

- **适用角色**：管理员、院长、教师（按权限看到本院或本人的导入记录）。
- **操作**：在「成绩管理」→「导入记录」或类似列表中，找到某次导入记录，点击「下载」，可下载当时上传的 Excel 文件备份。

## 四、试卷分析单上传

### 4.1 文件格式与大小限制

- **格式**：常见办公格式均可（如 Word、PDF、Excel 等），具体以系统上传页说明为准。
- **大小**：单文件一般不超过 **50MB**（与系统配置一致）。
- **命名**：建议文件名包含课程名、学期等，便于后续查找。

### 4.2 上传流程与权限

| 角色 | 上传 | 查看/下载 | 删除 |
|------|------|-----------|------|
| **教师** | 可上传 | 可查看、下载本人上传的 | 一般不可删除他人文件 |
| **院长** | 可上传 | 可查看、下载本学院教师上传的 | 一般不可删除 |
| **管理员** | 可上传 | 可查看、下载全部 | 可删除任意记录（用于纠错） |

**操作步骤（示例）：**

1. 登录后进入「试卷分析」或「试卷分析上传」菜单。
2. 点击「上传」，选择文件，填写备注（若有），提交。
3. 在列表页可按文件名、上传人筛选，点击「下载」可下载文件。

## 五、基本信息录入

### 5.1 谁可以录入什么

| 内容 | 录入方式 | 权限说明 |
|------|----------|----------|
| **学院、专业、年级、学期、班级、学生、教师、课程、开课、培养方案** | 仅**管理员**可进行新增、修改、删除、批量导入 | 教师、院长只能查看，不能增删改 |
| **学生** | 管理员：单个新增/编辑、批量导入（Excel） | 导入模板列为：学生学号、学生姓名、初始密码、班级名称、年级名称、专业名称、学院名称 |
| **教师** | 管理员：单个新增/编辑、批量导入 | 模板列：教师工号、教师姓名、初始密码、角色、所属学院名称 |
| **课程、开课、培养方案** | 管理员：单个或批量导入 | 模板中均为「名称」或「工号/学号」，无 ID |

### 5.2 录入顺序建议

为避免导入失败，建议按以下顺序维护基础数据：

1. 学院 → 2. 年级 → 3. 学期 → 4. 专业 → 5. 班级 → 6. 教师 → 7. 课程 → 8. 培养方案 → 9. 开课（教学管理）→ 10. 学生  

之后再进行成绩单上传、成绩发布等操作。

## 六、操作日志的使用

### 6.1 查看日志

- **适用角色**：管理员可查看全部操作日志；院长、教师一般可查看与本学院或本人相关的操作（以实际菜单为准）。
- **入口**：管理端「操作日志」或「日志管理」菜单。
- **内容**：操作者、操作类型、目标表、目标 id、操作内容、时间等。

### 6.2 过滤和搜索

- 可按**操作者**、**操作类型**、**时间范围**等条件筛选，便于追溯某次成绩导入、某条成绩修改或某次审批。
- 若支持按「目标表」或「关键词」搜索，可快速定位成绩、申请、学生等相关操作。

## 七、系统使用顺序 / 流程

### 7.1 信息录入顺序（管理员）

```
学院 → 年级 → 学期 → 专业 → 班级 → 教师 → 课程 → 培养方案 → 开课表 → 学生
```

- **先录入基础数据，再上传成绩单**。  
- 成绩单里的「课程名称」「开课学期」「任课教师」「开课班级」必须在系统中已存在，且与 Excel 中完全一致。

### 7.2 成绩相关流程概览

```
1. 教师/院长/管理员上传成绩单（Excel）
        ↓
2. 成绩进入系统，状态为「已上传」（学生不可见）
        ↓
3. 管理员在「成绩管理」中做「发布成绩」
        ↓
4. 成绩状态变为「已发布」，学生可查询
        ↓
5. 若需修改已发布成绩，走「成绩修改申请」流程（见下节）
        ↓
6. 管理员可将已发布成绩「锁定」，锁定后仅管理员可强制修正
```

### 7.3 审批流简要说明

- **成绩修改申请**：由教师或院长发起，填写申请原因、原成绩与修改后成绩，可选上传证明附件。
- **院长审批**（若启用）：院长可对本学院相关申请进行「通过」或「拒绝」；通过后状态为「院长已通过」，等待管理员终审。
- **管理员终审**：管理员对申请进行「通过」或「拒绝」。通过后，系统自动更新成绩并写入审计日志；拒绝时需填写拒绝原因。

## 八、成绩修改流程

### 8.1 流程说明（教师/院长提交申请 → 管理员审批 → 写入日志）

1. **谁可以提交申请**  
   - 教师：仅能对自己任课课程的成绩提交修改申请。  
   - 院长：仅能对本学院课程的成绩提交修改申请。  
   - 管理员：可对全部成绩提交申请（一般用于纠错或特殊审批）。

2. **何时可以申请**  
   - 仅当成绩状态为**「已发布」**时，才能对该条成绩提交修改申请；「已上传」或「已锁定」状态需先发布或由管理员处理。

3. **申请内容**  
   - 选择要修改的成绩记录，填写**原成绩**与**修改后成绩**（平时、期中、期末、总成绩等），填写**申请原因**，可选上传**证明附件**。

4. **审批与结果**  
   - **院长一级审批**（若启用）：院长对本学院申请进行通过/拒绝；通过后进入管理员终审。  
   - **管理员终审**：管理员通过后，系统自动更新成绩表并写入**成绩变更日志**，便于审计；拒绝时需填写拒绝原因，申请人可查看。

5. **日志与追溯**  
   - 所有成绩变更（含导入、发布、修改申请通过后的修改）都会写入**成绩审计日志**，不可删除，便于事后追溯。

## 九、成绩状态说明

### 9.1 三种状态

| 状态 | 说明 | 学生是否可见 | 是否可提交修改申请 |
|------|------|--------------|----------------------|
| **已上传（UPLOADED）** | 成绩单刚导入，尚未发布 | 不可见 | 不可 |
| **已发布（PUBLISHED）** | 管理员已「发布成绩」，正式生效 | 可见 | 可以 |
| **锁定（LOCKED）** | 管理员已锁定，不允许普通修改 | 可见 | 不可（仅管理员可强制修正） |

### 9.2 状态流转（管理员操作）

```
已上传 → [管理员发布] → 已发布 → [管理员锁定] → 锁定
```

- **发布**：将选中的「已上传」成绩批量改为「已发布」，学生即可查询。  
- **锁定**：将「已发布」成绩批量改为「锁定」，防止再被普通修改申请改动；仅管理员可通过「强制修正」功能修改锁定成绩。

## 十、常见问题（FAQ）

| 问题 | 处理建议 |
|------|----------|
| 忘记密码 | 联系管理员重置；若系统支持「忘记密码」，按页面提示操作。 |
| 学号/工号登录失败 | 确认使用学号或工号，而非数字 ID；确认用户类型选择正确；确认账号未停用。 |
| 成绩单上传失败：课程/学期/教师不存在 | 检查 Excel 中课程名称、学期名称、任课教师与系统完全一致；先在「教学管理」中维护开课信息。 |
| 成绩单上传失败：学号不存在 | 检查学生是否已导入，学号是否与 Excel 一致（无多余空格）。 |
| 成绩单上传失败：该学生本学期本课程成绩已存在 | 不能重复导入同一条成绩；需走「成绩修改申请」流程。 |
| 学生看不到成绩 | 确认该批成绩已被管理员「发布」为「已发布」；学生只能看到「已发布」状态。 |
| 无法提交成绩修改申请 | 确认该条成绩状态为「已发布」；确认当前角色有权限申请该课程（教师任课/院长本院）。 |
